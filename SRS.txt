Software Requirement Specification (SRS)
專案名稱：Task Timer & Reward App

（任務時間管理與獎勵系統）

1. 🎯 Purpose（目的）

本系統提供使用者建立「主任務 + 子任務」的時間管理機制，透過倒數計時、語音提示與完成紀錄，幫助使用者有效完成日常任務，並透過獎勵機制提升執行動機。

2. 👤 User Roles（使用者角色）
角色	說明
一般使用者	執行任務、查看紀錄與獎勵
管理者（受密碼保護）	建立主任務與子任務


* **程式語言**：HTML（可搭配 JavaScript / CSS）
* **目標族群**：國小學生

3. 🧩 System Overview（系統架構）

系統包含以下模組：

任務管理模組（Task Management）
任務執行模組（Execution Engine）
語音提示模組（TTS）
時間紀錄模組（Tracking）
獎勵系統（Reward System）
安全控管（Password Protection）
4. 📋 Functional Requirements（功能需求）
4.1 任務建立（Task Creation）
4.1.1 主任務建立
使用者需輸入密碼 1234 才可進入建立畫面
可建立多個「主任務」
4.1.2 子任務建立（TODO List）
使用者逐筆輸入：
任務名稱（string）
預計時間（秒 or 分鐘）

📌 Example：

刷牙 - 7秒
穿襪子 - 10秒
4.1.3 任務儲存
所有輸入需儲存於本地資料庫（建議：Room / SQLite）
主任務下可包含多筆子任務（1:N）
4.2 主頁顯示（Home Screen）
顯示所有「主任務列表」
每筆主任務顯示：
任務名稱
預計總時間
「開始」按鈕
4.3 任務執行（Execution Flow）
4.3.1 任務啟動
點擊「開始」後進入執行頁面
4.3.2 子任務執行流程（Sequential Execution）

系統需依序執行子任務：

Step 1：語音提示（TTS）

播放：

請在 X 秒內完成任務（刷牙），完成後請點擊完成
Step 2：計時開始
顯示：
任務名稱
倒數或累計秒數（進行中）
Step 3：完成按鈕
使用者點擊「完成」
系統記錄：
Start Time
End Time
Duration（實際耗時）
Step 4：自動進入下一個子任務
4.3.3 任務完成

當最後一個子任務完成時：

系統需計算：

總預計時間
實際總時間
差異時間（提前 or 延遲）
4.4 任務結果判定
條件	結果
實際時間 ≤ 預計時間	通過
實際時間 > 預計時間	未通過
4.5 獎勵系統（Reward System）
4.5.1 得分機制
每完成一個主任務且「在時間內完成」
→ 獲得 1 分
4.5.2 獎勵頁面

顯示：

每次任務紀錄：
日期
主任務名稱
開始/結束時間
是否通過
獲得分數
4.5.3 獎勵兌換設定
使用者可設定：
X 點 = 獎勵（例如：看電視30分鐘）
